Two Data Sources

ZZX Bitnodes Global Registry vs Bitnodes Original

ZZX Source

ZZX Bitnodes Global Registry

ZZX Bitnodes Global Registry is ZZX-Labs R&D’s own persistent crawler and static-API registry system. It is designed to maintain long-running state, preserve historical records, export static JSON files, track rolling reachable and unreachable windows, support archive replay, enrich records with GeoIP and ASN metadata, and generate frontend widgets for node intelligence analysis.

Original-Compatible Source

Bitnodes Original

Bitnodes Original is included as an original-compatible mode in honor of Addy Yeow’s work. Its purpose is to preserve the spirit and public API shape of the original Bitnodes crawler model so others can continue using and studying a familiar Bitnodes-style data source through this mirror.

Important Distinction

Not Identical Systems

ZZX Bitnodes Global Registry and Bitnodes Original are not exactly the same. They vary in how they crawl, retain state, replay archives, export snapshots, classify records, and present analytics. They should be understood as related but distinct crawler modes: one is ZZX-Labs’ extended registry, while the other exists to preserve original Bitnodes-compatible behavior and attribution.

Mirror Purpose

Preservation, Attribution, and Public API Continuity

Preservation

Keeping Bitnodes Useful

This mirror is intended to help Bitnodes-style network visibility live on for researchers, node operators, developers, students, and public Bitcoin infrastructure watchers. The goal is continuity: preserving a useful public model while extending it into a static frontend and independent crawler system.

Respect

Authorship Is Preserved

This mirror is not intended to erase authorship, obscure origin, or claim invention of the Bitnodes project. The original Bitnodes concept and its historical implementation belong in the credit record with Addy Yeow clearly named. ZZX-Labs R&D’s work begins from that inspiration and builds a separate mirror and registry implementation.

Public API

Open Data Shape

The mirror publishes static JSON API files for frontend use and public inspection. Where possible, ZZX-Labs R&D preserves familiar Bitnodes concepts such as latest snapshots, node lists, countries, cities, ASNs, agents, ports, versions, Tor nodes, propagation, peer health, and DNS seeder views.

ZZX-Labs Additions

Extended Mirror Functionality

Frontend Reconstruction

ZZX-Labs R&D extends the mirror with its own frontend styling, static JSON publication model, GitHub Pages compatibility, source-selection dropdowns, local crawler exports, website widget integration, and modular JavaScript and CSS architecture.

Telemetry Expansion

The mirror includes additional views for countries, cities, ASNs, Tor, latency, propagation, agents, ports, versions, service bits, VPN and datacenter heuristics, GeoIP enrichment, peer health, reachable-state history, and crawler source comparison.

Registry Engineering

The ZZX Bitnodes Global Registry is designed around persistent crawler state, rolling 24-hour reachability windows, private registry backups, chunked master archives, source-specific exports, and static JSON APIs suitable for public frontend mirrors.
