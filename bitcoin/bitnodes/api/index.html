Recommended Usage

Integration Notes

Use source-scoped APIs

Prefer api/zzxbitnodes/latest.json, api/zzxbitnodes/nodes.json, and api/aggregate/zzxbitnodes/latest.json for new tools.

Avoid tiny per-item directories

High-cardinality directories such as providers, organizations, nodes, and ASNs should consolidate into larger indexed JSON files instead of thousands of small files.

Use map-ready files for maps

Map pages should consume maps/*/points.json, maps/*/nodes.geojson, and live-map/*/live-map.json instead of parsing full snapshots.

Registry backups stay separate

Large global registry chunks belong in the registry backup workflow, not in the public API page directories.
