Tools

The ZZX-Labs Tools hub collects the public, internal, research, staff, automation, dashboard, API, data export, and technical utility systems used across the site. These tools support Bitcoin infrastructure monitoring, Bitcoin price intelligence, CyberChef-style data operations, mempool analysis, global reference data, site maintenance, research workflows, and future staff operations.

Some tools are public-facing. Some are staff-only. Some are mirrors or integrations of external open-source projects. Some are ZZX-Labs-specific wrappers, exporters, dashboards, and data pipelines designed to feed widgets, pages, reports, APIs, datasets, and research systems.

Public / Staff Split

Access Model

Tools should be separated by access level. Public tools can be safely exposed to visitors. Staff tools should require authentication or remain offline/local. Internal tools should never leak private credentials, administrative paths, payment secrets, unpublished datasets, or privileged system controls.

Security

Tool Safety Principles

Read-Only By Default

Public tools should prefer static, read-only data and avoid exposing private control paths or server-side authority.

No Secrets In Static Files

Never publish API keys, private tokens, admin URLs, payment secrets, credential material, or private backend paths.

Safe Exports

Export scripts should sanitize, validate, normalize, and minimize data before publishing to public pages.

Versioned Outputs

Important tool outputs should include timestamps, versions, source notes, hashes, and update cadence where practical.

Public / Private Split

Staff tools, internal dashboards, admin APIs, and sensitive workflows should stay separate from public static tooling.

Responsible Use

Security, OSINT, data, and transform tools should be used for lawful, authorized, defensive, research, and educational purposes.
