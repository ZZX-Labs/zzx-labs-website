Designing ZZX-Labs

This page outlines the artistic and engineering process used to craft the visual identity, color palette, grid layout, and frontend/backend architecture of zzx-labs.io. Everything here is intentional: from font kerning to shadow radius.

It also serves as a reference for future brand and UI/UX expansion across our mirror services, onion services, admin panels, and internal tools.

Design Elements & Concepts

Below are the key visual, typographic, and structural components that shape the ZZX-Labs brand identity. Each is interactive, documented, and tied to our style system or `*.json` declarations.

Color Palette

Core brand colors for ZZX-Labs, defined in static/colors.json. All buttons, hover states, and themes derive from this core swatch group.

Font Stack

Primary typefaces include the mono-optimized IBM Plex Mono and our custom header typeface Adult Swim Font. These are defined in static/fonts.json and used site-wide.

Animation Classes

Scroll-triggered animations use CSS utility classes. These fire when elements enter the viewport.

.scroll-animation {
    opacity: 0;
    transform: translateY(20px);
    transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}
.scroll-animation.in-view {
    opacity: 1;
    transform: translateY(0);
}

Inspirational Color Grid (Shuffle Algorithm)

Our grid of philosophical influencers uses color classes that are randomly shuffled on load to avoid adjacent duplicates.

// Pseudo-Random Fisher-Yates shuffle
function shuffle(array) {
  for (let i = array.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [array[i], array[j]] = [array[j], array[i]];
  }
}

Radio Stations

ZZX-Labs provides curated streaming `.m3u` radio playlists in static/audio/radio/. These feeds are used internally for lab ambiance and broadcast automation.

Soundtrack / Tracks

ZZX-Labs original music compositions, ambience loops, and soundtrack elements are stored in static/audio/tracks/. These can be embedded across the frontend or streamed via player.

Alert Sounds

All alert sounds for ZIRA, terminals, admin panels, and analytic dashboards are stored in static/audio/alerts/. Default format is 320kbps MP3.

Static Video Assets

All `.mp4` videos embedded or used in visual media across ZZX-Labs are stored in static/video/videos/. These are also used for OBS overlays and archival clips.

Live Video Feeds

Live stream feeds are served over a WebRTC or HTTP-based stream routed from local infrastructure. Used for admin observation, studio cams, or livestream integration.
