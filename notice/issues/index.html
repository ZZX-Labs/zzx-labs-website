Issues & Work Items

This page tracks larger site issues, repair tasks, migration work, feature requests, design tasks, missing pages, workflow gaps, and implementation items affecting the ZZX-Labs R&D website.

Known bugs are defects. Issues are broader work items. One issue may include multiple bugs, tasks, subtasks, decisions, dependencies, and release notes.

Completed work that should be documented in release notes.

Issue Types

What Counts As An Issue

Bug

A defect, broken path, script failure, rendering error, or incorrect behavior.

Task

A known work item needed to complete a page, system, script, or workflow.

Enhancement

A proposed improvement that expands usefulness, clarity, design, or automation.

Migration

Moving legacy pages, styles, content, data, or scripts into the new architecture.

Missing Page

A linked page, profile, category, tool, or documentation section that does not exist yet.

Decision

An unresolved architecture, naming, security, design, legal, or content decision.

Priority Model

Issue Priority Levels

P0 — Critical

Security, privacy, payment, data integrity, major site breakage, or operational risk.

P1 — High

Major broken pages, missing core content, damaged navigation, important tool failure, or severe workflow break.

P2 — Medium

Important but nonblocking work: incomplete content, missing subpages, partial automation, or styling repairs.

P3 — Low

Polish, spacing, copy cleanup, noncritical enhancements, optional links, and minor refinements.
