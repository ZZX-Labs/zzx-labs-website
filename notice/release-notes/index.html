Release Policy

How Releases Are Recorded

ZZX-Labs release notes should describe meaningful changes rather than every minor copy edit. A release note may cover a full page rewrite, a new page family, a major design migration, a script addition, a JSON data structure, an issue fix, or a new operational workflow.

Future releases should include dates, versions, issue references, affected pages, status, related files, known regressions, and follow-up work.

Current Release

Development Series 0.1 — Unified Page System

The current release series focuses on rewriting legacy pages into a unified layout and expanding minimal content into useful public-facing portals.

  1. 0.1.1 Core layout migration started.
    Top-level pages are being migrated to zzx-page-shell, zzx-page-hero, zzx-panel, zzx-panel-grid, and zzx-card-grid.
  2. 0.1.2 Donation and donor pages expanded.
    Donate page now explains Bitcoin funding, large donation coordination, donor recognition, and privacy. Donors page now functions as inactive opt-in recognition registry.
  3. 0.1.3 Contact key-history system designed.
    Added design for active, transitional, retired, discontinued, and historical GPG/software signing key records under contact/key-history/.
  4. 0.1.4 Incubation portal expanded.
    Incubation page now includes mission, operating model, pipeline, lifecycle stages, tracks, resources, funding, startup preview, and FAQ.
  5. 0.1.5 Startup ecosystem directory structured.
    Startup directory now groups ventures by category and supports future JSON-generated entries through category-aware dynamic insertion points.
  6. 0.1.6 Startup profile template created.
    incubation/startups/_template/index.html created as canonical startup/company profile template.
  7. 0.1.7 Startup submission workflow drafted.
    Python GUI/CLI workflow created to submit startup records, write JSON, generate subpages, validate slugs, and render template-based profiles.
  8. 0.1.8 Staff and partners upgraded.
    Staff page now includes pseudonymous profile cards and disclosure levels. Partners page now includes active, foundational, and future partner records.
  9. 0.1.9 Opportunities pages expanded.
    Careers, partnerships, investments, and volunteer pages expanded into proper entry points for contributors, partners, investors, and collaborators.
  10. 0.1.10 Notice system created.
    Notice hub and subpages created for status, construction, development, known bugs, issues, roadmap, progress, release notes, notes, and comments.

Known Regressions

Current Release Limitations

  • Some subpages are linked before they exist.
  • Generated records require JSON files that may not exist until initialized.
  • Some dynamic tools require local data mirrors or APIs that are still under development.
  • Some legacy pages may not yet match the unified layout.
  • Some profile pages are templates or placeholders rather than final public profiles.
  • Payment, account, shop, service-retainer, and API-toll systems are not final.
