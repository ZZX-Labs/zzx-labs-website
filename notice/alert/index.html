Active Alerts

This page is reserved for urgent ZZX-Labs R&D alerts, including active security warnings, phishing notices, fraudulent payment address reports, service outages, signing-key emergencies, infrastructure incidents, release rollbacks, DNS issues, certificate events, and time-sensitive public warnings.

Alerts are different from breach notices. A breach notice documents a confirmed incident. An alert warns users about an active or time-sensitive condition that may require immediate attention.

Alert Types

What Belongs Here

Security Warning

Active vulnerability, exploit attempt, malicious campaign, suspicious release, or immediate security concern.

Phishing / Impersonation

Fake accounts, spoofed emails, fraudulent domains, impersonation attempts, or malicious contact requests.

Payment Warning

Fraudulent Bitcoin address, outdated donation address, payment workflow change, or suspected payment redirection.

Signing-Key Emergency

Immediate key retirement, suspected key compromise, replacement fingerprint, or emergency release verification update.

Service Outage

Major interruption affecting public pages, tools, APIs, widgets, mirrors, or future private services.

Infrastructure Event

DNS, TLS, hosting, server, mirror, GitHub Pages, certificate, routing, or deployment emergency.

Severity

Alert Severity Levels

Critical

Immediate user action may be required. Examples: compromised signing key, fraudulent payment address, active exploit, or confirmed dangerous release.

High

Important security or reliability issue. Users should review the alert before using affected systems.

Medium

Noticeable impact or security concern, but no immediate emergency for most users.

Low

Advisory notice, planned maintenance, degraded service, or noncritical warning.

Emergency Verification

How Users Should Verify Alerts

  1. 01Check this page: Confirm whether an alert is active or resolved.
  2. 02Check key history: For signing-key events, verify against contact/key-history/.
  3. 03Check release notes: Confirm whether a release was updated, rolled back, or replaced.
  4. 04Check payment pages: For Bitcoin payment warnings, verify donation and service payment addresses before sending funds.
  5. 05Contact ZZX-Labs: For urgent uncertainty, use official contact paths rather than third-party claims.
