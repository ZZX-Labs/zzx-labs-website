Known Bugs & Regressions

This page tracks known bugs, broken links, incomplete features, widget failures, API gaps, layout regressions, missing assets, generated-content issues, and other defects affecting the ZZX-Labs R&D website during active development.

A listed bug does not mean the site is abandoned. It means the issue is known, visible, and pending review, repair, redesign, or replacement.

Known Bug Categories

What Gets Tracked Here

Broken Links

Subpages that are referenced before their directories or files are created.

Missing Assets

Images, logos, icons, fonts, audio, video, scripts, styles, or JSON files not yet present.

Widget Issues

Bitcoin widgets, ticker panels, Bitnodes, mempool, mining stats, BPI, and future dashboards.

JavaScript Errors

Script loading, path resolution, injected header/footer, generated records, and dynamic data issues.

Layout Regressions

Mobile spacing, card grids, panels, image sizing, typography, overflow, and responsive behavior.

Data Issues

Missing JSON, outdated records, bad schemas, duplicate slugs, stale API data, and malformed values.

Current Known Bugs

Open Defects

  1. OPEN Some profile subpages are linked before creation.
    Staff, partner, consultant, startup, career, and project profile pages may have links before their individual index.html files are generated.
  2. OPEN Some widgets depend on local data not yet deployed.
    Bitcoin, Bitnodes, mempool, mining, and data-dashboard widgets may require JSON snapshots, APIs, or mirrors that are still under construction.
  3. OPEN Generated startup records require data file availability.
    The startup directory dynamic renderer expects incubation/startups/_data/startups.json. If absent, generated records will not appear.
  4. OPEN Blog workflows are not final.
    Blog scheduling, RSS, post repositories, generated publication pages, and random publishing workflows still need implementation.
  5. OPEN Project-derived pages are not yet fully automated.
    Portfolios, downloads, documentation, papers, and related views should eventually be generated from project records, but this system is still pending.
  6. OPEN Payment/account workflows are placeholders.
    Service payments, user accounts, shop flows, API tolls, retainers, and Lightning payment workflows are not final and require security design before production use.

Severity Model

Bug Severity Levels

Critical

Security risk, payment risk, data loss, broken core navigation, exposed private information, or severe site failure.

High

Major broken page, nonfunctional tool, missing critical dependency, inaccessible important section, or serious path error.

Medium

Partial feature failure, layout issue, missing asset, incomplete data, noncritical broken link, or degraded experience.

Low

Minor copy issue, small spacing defect, minor image issue, low-impact link issue, or incomplete enhancement.

Cosmetic

Visual polish, alignment, colors, spacing, typography, card balance, or presentation refinement.

Enhancement

Not a bug, but a requested improvement, future feature, workflow upgrade, automation task, or design enhancement.

Reporting Format

How To Report A Bug

Useful bug reports are specific. Include the affected URL, what you expected, what happened, whether the issue affects layout, content, links, JavaScript, data, API behavior, assets, or mobile rendering, and any visible console error if available. 

Bug Report
├── Page URL
├── Bug Title
├── Expected Behavior
├── Actual Behavior
├── Browser / Device
├── Severity Estimate
├── Steps To Reproduce
├── Screenshot / Error Text
└── Suggested Fix
