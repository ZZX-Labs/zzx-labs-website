VidGhee

“Clarify” your videos: batch transcode, denoise, deinterlace, stabilize, trim, and optimize with reproducible recipes.

What it is

VidGhee is a practical, deterministic video processing toolkit for turning messy captures into clean, storage-efficient masters. It wraps ffmpeg, vapoursynth/avisynth-style filters, and optional GPU acceleration into named recipes you can run locally or on a node farm. Recipes are declarative (YAML/TOML/JSON), versioned, and portable across machines.

Recipe Engine

Reproducible pipelines: trim → denoise → scale → stabilize → encode → tag.

Smart Presets

H.264/H.265/AV1, grain-preserve, animation, HDR-aware, mezzanine (ProRes/DNxHR).

GPU Accel

NVIDIA NVENC, Intel QSV, Apple VTB; CPU x264/x265 fallback with tuned CRF/bitrate.

Audio Care

Loudness norm (EBU R128), channel mapping, dialogue enhance, format-aware tagging.

Integrity

Frame/hash manifests, deterministic settings, job logs, and artifact checks.

Batch + Watch

Point at a folder, get clean output; optional watch mode and web UI runner.

CLI (example)

vidghee run \
  --src "/captures/*.mp4" \
  --dst "/masters" \
  --recipe "clarify-h265" \
  --gpu nvenc --crf 20 --audio eac3_5_1

Recipe (YAML)

# recipes/clarify-h265.yaml
steps:
  - trim: { start: "00:00:02.000", end: "end-00:00:02.000" }
  - yadif: { mode: "send_frame" }         # deinterlace if needed
  - denoise: { preset: "knlmeans-medium" }
  - stabilize: { mode: "deshake-lite" }
  - scale: { width: 1920, height: 1080, algo: "lanczos" }
encode:
  video: { codec: "hevc", preset: "slow", crf: 20, tune: "grain" }
  audio: { codec: "eac3", bitrate: "384k", channels: "5.1" }
  container: mp4
artifacts:
  sidecar: ["json","md"]
  hashes: ["sha256","blake3"]
