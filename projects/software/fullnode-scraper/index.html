fullnode-scraper

Bitcoin full node peer discovery, scraping, and analytics. Resolve peers via DNS seeds, P2P addr/addrv2 gossip, and targeted crawls—then enrich with ASN/Geo, versions, service flags, and uptime signals.

Open Project View on GitHub

Research-focused. Respects rate limits and configurable ethics guardrails. Tor/I2P optional. Store as SQLite/Parquet for reproducible studies.

Overview

fullnode-scraper crawls the Bitcoin network graph to maintain a current snapshot of reachable peers. It seeds from DNS, passive gossip (addr/addrv2), and optional targeted probes, recording observed endpoints and traits such as user agent, protocol version, service bits (e.g., NODE_NETWORK, NODE_WITNESS), last seen, handshake latency, and open transport(s). Datasets are normalized for longitudinal analysis.

        Highlights

        • Multi-seed discovery: DNS seeds → P2P addr/addrv2 gossip → optional focused scans (configurable).
        • Enrichment: ASN, GeoIP (country/region), reverse DNS, service flags, UA, proto version, uptime hints.
        • Respectful crawling: concurrency caps, jittered backoff, allow/deny lists, Tor routing, and pause windows.
        • Structured output: SQLite for quick queries; Parquet/CSV for data science; daily snapshots with hashes.
        • Time-series ready: rollups for version adoption, relay capability trends, IPv4/IPv6 mix, region distribution.
        • APIs & CLI: local REST viewer, export commands, and notebook-friendly schema.
        • Reproducible: config-locked runs, signed manifests, and provenance metadata per snapshot.

        Ethics & Safety

        This tool is for measurement and research. Default settings avoid aggressive probing and support Tor/I2P routing. Always comply with local law and community norms; do not DoS or harass peers. Use conservative concurrency and respect opt-out lists. Disable targeted scans if you merely need DNS+gossip coverage.
