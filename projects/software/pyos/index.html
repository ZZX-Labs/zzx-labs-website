PyOS

A Python-first operating system (32/64-bit) aiming for broad app portability through layered compatibility, sandboxed runtime, and declarative packaging.

Open Project GitHub Hugging Face

What is PyOS?

PyOS is an experimental “Python-native” OS concept exploring how far a modern system can go with Python as the primary coordination layer. The vision is a modular environment with a microkernel-style core, userland services in Python, and compatibility layers that enable running diverse software stacks via shims, containers, and emulation where appropriate.

Note: The ambitious goal of running software built for Windows, Linux, macOS, Android, or iOS “natively and interchangeably” is a long-term research direction. In practice, PyOS uses compatibility layers, containers, or emulation depending on the target app and platform.

Key Capabilities

Python-First Runtime

System services, package management, and most userland tools are orchestrated in Python for rapid development and readability.

32/64-bit Targets

Build profiles for 32-bit and 64-bit devices with per-arch wheels and optional native extensions.

Compatibility Layers

Run diverse software via shims (POSIX/Wine/proot), containers (OCI), or VMs. Declarative manifests select the best strategy.

Sandbox & Permissions

Profiles for filesystem/network/capability isolation; reproducible environments and signed manifests.

Declarative Apps

Apps describe dependencies and launch surfaces in a JSON/YAML manifest; PyOS resolves, builds, and runs them.

UI Options

CLI-first with optional GUI shells (Qt/PySide) and web UIs for remote management.
