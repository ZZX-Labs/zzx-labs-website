GP — Personal Digital AI Assistants

Modular, multi-modal GPT assistants (text • image • audio • TTS • STT) with SFW and NSFW variants, privacy-first ops, and optional on-device inference.

Open Project View on GitHub

GP is an orchestration layer for building domain-tuned assistants and agents. SFW/NSFW policies are explicitly separated and operator-configurable.

Overview

GP provides a configurable framework for Personal Digital Assistants leveraging large language and multi-modal models. It supports prompt routing, tool access (search, code exec, file I/O), voice I/O, and memory backends with strict privacy controls. Deploy as local-only, hybrid (local + API), or fully managed. SFW and NSFW capability sets are isolated via policy modules.

        Highlights

        • Multi-modal: Text, image gen/edit, audio I/O (TTS/STT), and basic video workflows.
        • Profiles: SFW/NSFW variants with distinct safety policies and capability gates.
        • Privacy-first: Local-first memory, encrypted stores, and offline-capable pipelines.
        • Tooling: Skills framework (web fetchers, code runners, RAG, calendar/mail hooks, etc.).
        • Routing: Model-agnostic orchestrator with adapters for local and hosted LLMs.
        • Voice: Real-time VAD, hotword wake, low-latency TTS, and diarization options.
        • Ops: Prompt packs, audit trails, red-team tests, and policy snapshots per profile.

        Safety & Policy

        NSFW features are isolated in separate profiles with explicit operator opt-in and jurisdiction checks. Content filters, consent gates, logs, and rate-limits are provided. Operators are responsible for local compliance and end-user disclosure. SFW defaults ship enabled; NSFW is disabled by default.
