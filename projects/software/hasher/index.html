Forensics-grade: resumable scans, merklized chunks, and signed manifests.

Overview

Hasher is a forensic-oriented hashing system for whole disks, RAID/NAS arrays, and large file trees. It computes multi-algorithm digests (BLAKE3/SHA-256/SHA-512/xxHash), supports fixed or adaptive chunking, builds Merkle trees for any object, and writes an append-only, tamper-evident record to a local database. Results are queryable by path, inode, size, time window, algorithm, or content hash.

        Highlights

        • Whole-device & tree hashing: block devices, LVM/RAID, network mounts, and folders.
        • Resumable & parallel: checkpointing, multi-threaded IO, back-pressure aware.
        • Chunking & Merkle: fixed/adaptive chunks, per-file or per-device Merkle roots.
        • Multi-algo: BLAKE3, SHA-256, SHA-512, xxHash; add your own via plugins.
        • DB-backed catalog: SQLite (local) or Postgres (shared) with signed manifests.
        • Dedup & similarity: rolling hashes + optional fuzzy features for near-dupe hints.
        • CLI & service modes: one-shot commands or long-running agent with API.
        • Export: JSONL/CSV/Parquet manifests; SRI strings; SBOM-adjacent inventories.

        Quick Demo

        hasher scan /mnt/array \
  --algos blake3,sha256 \
  --chunk 4MiB --resume \
  --db sqlite:///hasher.db \
  --out manifest.jsonl

        Run locally; adjust mount points and database URL for your environment.
