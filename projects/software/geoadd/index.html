Open Project View on GitHub

Works offline with optional local tiles & gazetteers. Respects user privacy and supports reproducible, signed address records.

Overview

geoadd is a toolkit to normalize messy addresses, enrich them with coordinates, and emit compact spatial encodings (geohash, H3, and Open Location Code). It supports batch CSV/JSON processing, reverse geocoding, confidence scoring, and audit logs for compliance. Pluggable providers include local gazetteers (OpenStreetMap extracts), offline reverse indexes, or remote APIs you configure.

        Highlights

        • Normalization: Postal-aware parsing, casing, abbreviations, and canonical formats (per locale).
        • Geocoding: Multiple providers with priority/weighting; offline OSM-backed indexes supported.
        • Reverse geocoding: Coordinates → nearest address/POI with distance and confidence scoring.
        • Codes: Export geohash, H3 (res configurable), and Open Location Code (plus codes).
        • Batch & CLI: CSV/JSON in → enriched CSV/JSON out; streaming mode for large datasets.
        • Privacy & audit: Redaction rules, deterministic hashing of PII, signed manifests, reproducible runs.
        • Integrations: Simple REST server mode, local web UI, and library APIs for Node/Python.
