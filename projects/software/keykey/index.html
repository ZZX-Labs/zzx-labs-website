keykey — USB Hash Fingerprint Gate

A tiny file on a USB drive becomes a cryptographic “presence” signal. Systems check its fingerprint to unlock data arrays, wallets, or secure modes.

Open Project View on GitHub

Local-only; nothing leaves your browser. Works offline once loaded.

Overview

keykey is a simple, auditable mechanism: place a small file (e.g., device.keykey) on a USB stick. At boot or mount time, your system computes its hash and compares it to an allowlist (and optional policy). If matched, protected services or drives are enabled. The file may be opaque or structured JSON with policy hints; verification is always by hash.

        Implementation Notes

        • Presence via hash: hash of *.keykey is your proof of presence; contents can be random bytes or structured JSON.
        • Multi-factor: combine USB presence (hash match) with passphrase/PIN or TOTP (outside this demo).
        • Policy hints: if JSON, include e.g. allow (paths), expires, issuer, aud, and comment.
        • Auditability: keep an append-only log of accepted fingerprints and timestamps.
        • Rotation: support multiple current/next fingerprints to rotate keys safely.
