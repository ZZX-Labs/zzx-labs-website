VideoSort

Fast, deterministic video library sorting & tagging — by codec, resolution, audio, content, hashes, and rules.

What it is

VideoSort ingests scattered video files and places them into a clean, predictable folder structure. It extracts rich metadata (codec, bitrate, resolution, duration, audio layout, subtitles), computes cryptographic hashes, applies configurable naming templates, and writes tags/sidecars. Optional ML hooks can classify content, generate thumbnails and contact sheets, and detect duplicates or near-duplicates.

Deterministic Rules

Template-based renames and folders (e.g., /{year}/{codec}/{width}x{height}/).

Deep Metadata

FFprobe-backed extraction: codecs, streams, HDR flags, language tracks, chapters, more.

De-dupe & Hash

Multi-hash (SHA-256, BLAKE3) + perceptual frame hashing to spot near-duplicates.

ML Optional

Plug in local models for labeling, NSFW flags, OCR, and smart search indices.

Safe Moves

Dry-run previews, conflict resolution, atomic moves/copies, and rollback logs.

Exports

Write sidecars (JSON/CSV/NDJSON), media catalogs, and browsable HTML reports.

CLI (example)

videosort ingest \
  --src "/mnt/media/incoming" \
  --dst "/mnt/media/library" \
  --template "/{year}/{codec}/{width}x{height}/{basename}" \
  --hash sha256,blake3 --dry-run

Config

# videosort.config.toml
[ingest]
  template = "/{year}/{codec}/{width}x{height}/{basename}"
  write_sidecars = true
  sidecar_format = "ndjson"
  hashes = ["sha256","blake3"]
  move_mode = "atomic"

Integrations

- ffmpeg/ffprobe
- ImageMagick (contact sheets)
- Optional: OpenCL/CUDA for speed
- Optional: local ML models
