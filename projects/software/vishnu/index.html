Vishnu

Orchestrator for multi-pipeline workflows: schedule, fan-out/fan-in, retry trees, and resource-aware routing.

What it is

Vishnu coordinates DAG-style workloads across runners (e.g., Vikram) and services (e.g., VidGhee, VideoSort, GardenHarvester). Define jobs declaratively, wire dependencies, set SLAs and concurrency, and let Vishnu handle backpressure, retries, and idempotency. Emits compact JSON/NDJSON events consumable by Vishal for live visuals.

Declarative DAGs

YAML/JSON pipelines with parameterized tasks, fan-out/fan-in, conditions, and artifacts.

Smart Scheduling

Rate limits, quotas, resource tags, and queue priorities per tenant or pipeline.

Resilience

Retries with jitter, circuit breakers, idempotency keys, and dead-letter queues.

Pluggable Runners

Adapters for Vikram workers or custom executors (container, process, remote HTTP).

Observability

NDJSON event stream + metrics to feed Vishal dashboards and external monitoring.

Audit & Policy

Signed pipeline manifests, change logs, policy gates, and approvals where needed.

Run (server)

vishnu serve \
  --bind 0.0.0.0:7788 \
  --pipelines ./pipelines/ \
  --events ./out/events.ndjson

Minimal pipeline

# pipelines/transcode.yaml
pipeline: transcode
on:
  schedule: "*/5 * * * *"
steps:
  - id: probe
    uses: vikram:probe
    with: { input: "/ingest/*.mp4" }
  - id: encode
    needs: [probe]
    uses: vikram:encode
    with: { preset: "h264-720p" }
  - id: publish
    needs: [encode]
    uses: http:post
    with: { url: "https://cdn.local/api/publish" }

Webhook trigger

POST /trigger/transcode
