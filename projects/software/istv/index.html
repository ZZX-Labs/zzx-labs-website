ISTV — Internet Streaming TeleVision (Audio + Video)

A unified, decentralized network for 24/7 audio & video channels: signed manifests, swarm delivery, and smooth HTTP fallbacks.

Open Project View on GitHub

This page ships with a dependency-free demo player for both audio and video sources.

Overview

ISTV blends ISAN (audio) and ISVN (video): publish live or scheduled channels using signed manifests. Viewers fetch via peer swarms (WebRTC/QUIC) with HTTP/CDN fallback. This demo uses native HTML audio/video only.

        Architecture (Planned)

        • Publisher nodes: ingest (file/RTMP/WebRTC) → transcode (audio + video) → chunk → sign → announce.
        • Manifests: channel.json describes audio & video renditions, schedules, fallbacks, and policy.
        • Transport: Swarm (WebRTC/QUIC) with HTTP(S)/CDN fallback.
        • Signals: minimal telemetry, rotating peer IDs, privacy-first defaults.
        • Monetization: LNURL/WebLN tips, gated channels, PPV segments (opt-in).
