discord-downgrader

One-click audio size targeter for Discord’s <10 MB limit — trims, resamples, re-encodes, and normalizes so your clip just fits and still sounds good.

Open Project View on GitHub

Automatically chooses codec/bitrate/channel/sample-rate combos to land under a safe threshold (e.g., 9.5 MB). No guesswork.

Overview

discord-downgrader is a tiny tool for creators and community managers who share music clips, podcast excerpts, or voice notes on Discord. Drop in audio and get a processed output that reliably comes in under the upload cap — using smart heuristics (Opus/MP3/AAC), VBR with size targeting, optional mono-downmix, loudness normalization, and tail trimming/smart fade.

        Highlights

        • Size targeting: pick 10 MB (or custom), tool computes headroom (~9.5 MB) and tunes settings.
        • Smart codecs: Opus (webm/ogg) priority for best quality/size; MP3/M4A fallback for compatibility.
        • Auto resample: 48 kHz → 44.1/32/24 kHz when helpful; optional mono downmix for voice.
        • Loudness: optional normalization to a friendly target (e.g., ~-16 LUFS integrated) to avoid clipping.
        • Trim & fade: optional tail trim to hit target duration, with 10–50 ms safety fade.
        • Batch mode: drag multiple files; outputs named with codec/bitrate suffix.
        • Offline-first: runs locally; can bundle FFmpeg/avlib or use system binaries.
