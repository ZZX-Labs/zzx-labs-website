Vikram

Lightweight job runner & queue for reproducible pipelines. Files in, results out. Local or SSH workers.

What it is

Vikram is a file-first task runner and work queue for batch jobs and data-processing pipelines. Describe steps in YAML/TOML/JSON, point at an input folder, and Vikram handles discovery, retries, concurrency, and artifact tracking. Use it stand-alone or as the “glue” behind tools like VidGhee, VideoSort, and GardenHarvester.

Declarative Pipelines

Steps with inputs, outputs, env, timeouts, and “only if changed” guards.

Local + SSH Workers

Run jobs on this machine or fan out over SSH to a small farm of nodes.

Deterministic Artifacts

Sidecar manifests (JSON/MD), checksums, and resumable checkpoints.

Safe Concurrency

Named locks, rate limits, backoff/retry, and idempotent task design.

Observability

Structured logs, run history, job IDs, and an optional web dashboard.

Simple Ops

Drop a recipe file, run one command. No heavy queue broker required.

CLI (example)

vikram run \
  --src "/data/inbox/*.mp4" \
  --dst "/data/out" \
  --recipe "./recipes/cleanroom.yaml" \
  --concurrency 4 --retry 3

Recipe (YAML)

# recipes/cleanroom.yaml
discover:
  glob: "*.mp4"
  recurse: true
  hash: "sha256"
steps:
  - name: probe
    run: "ffprobe -hide_banner -v error -print_format json -show_format -show_streams {{src}} > {{dst}}/{{base}}.probe.json"
  - name: normalize-audio
    if_missing: "{{dst}}/{{base}}.norm.mp4"
    run: "ffmpeg -y -i {{src}} -c:v copy -c:a aac -b:a 192k {{dst}}/{{base}}.norm.mp4"
  - name: tag
    run: "python tools/tag.py {{dst}}/{{base}}.norm.mp4 --write '{{dst}}/{{base}}.tags.json'"
artifacts:
  sidecar: ["json","md"]
  hashes: ["sha256","blake3"]
