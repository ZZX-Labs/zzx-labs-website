Storage Policy

Rules for safe repository documentation and future service implementation.

Encrypt Private Data

Private storage should use encryption at rest, strict permissions, and separated user spaces.

Separate Public and Private

Public downloads and private vaults must never share writable paths or access-control boundaries.

Audit Uploads

All future uploads should be logged, scanned, staged, reviewed, and versioned before publication.

No Secrets in Git

Keep private paths, keys, encrypted payload indexes, tokens, backups, and dumps out of the repo.
