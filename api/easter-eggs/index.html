Easter Eggs API

The Easter Eggs API is ZZX-Labs' hidden interface layer, designed to reward exploration, inject mystery, and simulate secure badge-access puzzles or behavior-based triggers.

It responds to known secret phrases, paths, hashes, or specific user/session combinations with encrypted payloads, ASCII art, visual memes, sound FX, or system access hints. Often used in security testing, gamification, or internal training scenarios.

API Behavior

  • Hidden Routes: Triggers activated by obscure URLs, cookies, or headers.
  • Hash Unlocks: Users must POST or GET known hashes or phrases to unlock.
  • Visual/Audio Payloads: Encrypted art, audio, and response templates.
  • GPG Challenge: Solves puzzles using signature-based logic flows.
  • Timed Access: Payloads change depending on UTC or lunar phase.

Get a Random Encrypted Payload

Returns a base64 or hex-encoded message that must be decrypted using our public key.

curl https://zzx-labs.io/api/easter-eggs/random
{
  "payload": "-----BEGIN MESSAGE----- ...",
  "encoding": "GPG",
  "hint": "Try decrypting with ZZX public key"
}

Trigger an Egg by Phrase

POST or GET a known phrase that unlocks a specific egg entry. Requires signed headers for rare entries.

curl -X POST https://zzx-labs.io/api/easter-eggs \
  -H "X-Signature: user-gpg-sig" \
  -d 'trigger=zero-trust-is-the-new-black'

ASCII Art Response

Returns styled ASCII payloads used for terminals, login shells, or fun alerts. Output can rotate.

    • Badge Unlocks

    curl https://zzx-labs.io/api/easter-eggs/flag?user=0xdeadbeef
