CLI API (cmder Integration)

The CLI API bridges our command-line interface (CLI) tools with the ZZX-Labs backend ecosystem. It enables secure, remote execution of shell commands, API calls, and administrative actions from within a hardened terminal interface—integrated with our custom cmder environment.

This API supports real-time command execution, output capture, log streaming, access scoping, and two-way encrypted messaging between user and host environments. Designed for sysadmins, devops, analysts, and researchers working within the ZZX-Labs infrastructure.

Core Capabilities

  • Remote Shell Access: Issue and return terminal commands securely through signed JSON.
  • Command Profiles: Commands are grouped under access tiers and can be filtered or expanded per user.
  • GPG-AES Layer: All input/output is protected via end-to-end encryption using PGP or AES-256-GCM.
  • Output Streaming: View stdout, stderr, and logs in real time.
  • Sandboxed Exec: Commands are rate-limited and run in containerized environments with rollback options.

Run Command

Execute a terminal command over the authenticated API pipe. Returns stdout, stderr, and exit code.

{
  "cmd": "df -h",
  "auth": "gpg-signed-user-token",
  "context": "disk"
}

Command Sets

Chain together multiple commands or prebuilt scripts and submit as batch jobs. Useful for diagnostics, admin operations, or deploy routines.

{
  "script": [
    "uptime",
    "whoami",
    "ls -l /var/log"
  ],
  "user": "admin",
  "gpg_key": "xxxx"
}

Output Stream

View live command output from within the terminal, browser UI, or websocket-connected interface panel.

  • Live Output Stream

    • Command History

    Query all previously run commands, sorted by timestamp or user, with output previews and security labels.

  • Command History API

    • Access Class Enforcement

    All API-triggered commands are filtered by access tier, context ID, or task domain. This ensures scoped execution under role-based controls.

    {
  "role": "auditor",
  "allowed": [
    "cat /var/log/",
    "uptime"
  ]
}
