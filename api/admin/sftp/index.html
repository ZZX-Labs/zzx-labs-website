SFTP Server Control API

This endpoint allows admin-level control over ZZX-Labs' secure file server, which is hosted locally and accessed via SSH/SFTP. All transactions are authenticated using GPG-signed tokens and internal access tiering.

Key Functions

  • Mount Points: Dynamically mount and unmount file volumes for project-specific or department-specific access.
  • Permission Management: Set read/write access for each user, project, or token.
  • Credential Issuance: Issue time-bound or scoped login credentials for SFTP access.
  • Archive Sync: Push/pull archives to remote mirrors or encrypted vaults.

Mount Volume

Mount an encrypted LUKS container with specified passphrase and UUID. Must be pre-registered.

{
  "auth_token": "admin-tier-gpg-token",
  "command": "mount",
  "volume_id": "vault001"
}

Set Permissions

Define access rights for a file, folder, or entire volume. Supports recursive permissions and expiration timers.

{
  "command": "set_permissions",
  "target": "/mnt/sftp/project-alpha",
  "access": {
    "user": "bob",
    "permissions": "read-write",
    "expires": "2025-12-31T00:00:00Z"
  }
}
