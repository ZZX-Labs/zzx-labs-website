Recon Dashboard API

The Recon API provides tooling to detect, track, and profile users, bots, and crawlers interacting with ZZX-Labs systems. It distinguishes passive observers from active threat actors and escalates users through Tiers 0–5 based on behavioral triggers.

Designed for forensic visibility and threat simulation environments, this API supports honeypot injection, behavior sandboxing, and ZIRA-enhanced pattern recognition to trace lateral movement and persistence attempts.

Core Capabilities

  • Actor Fingerprinting: Fingerprint sessions by IP, User-Agent, navigation pattern, header entropy, or JS fingerprinting.
  • Recon Tiering: Promote flagged users through Tiers (1–5) based on behavior or access attempt severity.
  • Honeypot Triggers: Insert honeypot paths, buttons, inputs, or API endpoints to trigger traceable events.
  • ZIRA Sandboxing: Hand off unknown actors or command payloads to ZIRA for live sandboxed behavior simulation.
  • Behavioral Graph: Construct graphs of movement paths, dwell time, repeated probes, and payload injection vectors.

Tier Escalation Example

Manually or programmatically escalate a tracked user to a higher threat tier.

{
  "action": "escalate",
  "actor": "192.168.1.7",
  "tier": 4,
  "reason": "Multiple invalid token attempts + rapid endpoint cycling"
}

Inject Honeypot

Deploy a new honeypot path to trigger surveillance upon access.

{
  "inject": "honeypot",
  "path": "/admin-dashboard/settings/logs-export",
  "auth": "admin-gpg-token"
}
