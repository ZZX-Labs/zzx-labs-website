Secure Storage Server API

This administrative API allows you to interface with ZZX-Labsâ€™ encrypted local storage infrastructure. All files stored through this endpoint are locked using a combination of LUKS, AES-512, and optional GPG-layered encryption. Used for archiving research, mirroring critical data, and secure file segmentation.

Core Functionalities

  • Volume Control: Mount, unmount, or remount encrypted volumes tied to a research group, project, or system component.
  • Encryption Keys: Rotate LUKS headers, re-encrypt files, or attach GPG trust layers.
  • Storage Quotas: Assign quota limits per storage pool or project namespace.
  • Mirroring: Push or pull snapshots between storage pools and off-site vaults.
  • Access Logging: Monitor who accessed what, when, and from which interface.

Mount Encrypted Volume

Attach an encrypted volume by UUID and unlock with a passphrase + GPG auth layer.

{
  "cmd": "mount_volume",
  "volume_uuid": "vol-173f1b5e",
  "gpg_sig": "user-admin-keysig"
}

Push Mirror Snapshot

Mirror a compressed snapshot of the storage vault to a designated offsite host (SFTP or offline disk).

{
  "cmd": "mirror_push",
  "target": "cold-storage-02.zzx",
  "vault": "downloads_archive",
  "token": "admin-auth-token"
}
