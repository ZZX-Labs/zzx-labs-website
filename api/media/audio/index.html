Audio Stream Visualizer API

The Audio Stream API enables live and archived playback of audio feeds from ZZX-Labs’ infrastructure—supporting radio stations, audio alert systems, and musical archives via standardized endpoints.

Integrated with real-time visualizers (e.g., projectM, waveform renderers) and capable of delivering low-latency audio over HTTP, this API supports analytics dashboards, alert systems, and audio streaming panels.

Capabilities

  • Live Radio: Streams .m3u or .pls formatted playlists for FM/Net radio compatibility.
  • Alert Playback: Instant trigger of predefined alerts or warnings from static/audio/alerts/.
  • Track Library: Browse and play musical assets from static/audio/tracks/.
  • Visualizer Integration: Real-time FFT stream overlay support for projectM and canvas-based renderers.
  • Dashboard Audio: Auto-playable alerts and background loops in admin/control panels.

Live Radio Endpoint

Stream a radio playlist file directly or feed a live stream URL to an embedded player.

  • ZZX-Labs Radio

    • Alert Audio Playback

    System-wide alerts, warning sounds, and tones triggered in dashboards and terminals. Access these directly or trigger via API endpoint.

  • Browse Alerts

    • Music Track Library

    Preview internal track collections used in internal streams, demos, or soundtrack content. All tracks are 320kbps MP3 by default.

  • Browse Tracks

    • Visualizer Stream Output

    JSON endpoint returns audio data frames used by client-side visualizers to render waveforms, frequency maps, or spectrum animations.

    {
  "timestamp": "2025-05-01T14:00:00Z",
  "fft": [0.1, 0.32, 0.5, 0.4, 0.2],
  "peak": 0.95,
  "rms": 0.64
}
  • FFT JSON Feed

    • Secure Audio Upload

    Authenticated users can upload MP3 files to categorized folders (/tracks/, /alerts/, /radio/) using signed tokens.

    curl -F "file=@sound.mp3" -H "Auth: gpg-token" https://zzx-labs.io/api/media/audio/upload
