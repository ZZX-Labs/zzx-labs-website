Payments API

The Payments API enables all commerce, invoicing, and financial interaction across the ZZX-Labs ecosystem. Every transaction is conducted using Bitcoin (on-chain or Lightning Network), enforcing full sovereignty and zero reliance on fiat infrastructure.

All payments are assigned unique, one-time Bitcoin addresses generated from ZZX-Labs' self-hosted full node. Each transaction is tracked, logged, and recorded with full audit trail metadata.

Create New Payment

Initiates a payment request and generates a fresh Bitcoin address (P2WPKH or P2TR) for the transaction. Metadata includes description, purpose, satoshi value, and expiration timer.

  • /api/payments/create

    • Invoice Generator

    Generates a signed Bitcoin invoice (BIP-70 compatible) or BOLT11 invoice for Lightning transactions. Includes optional label, memo, expiration, fallback addresses.

  • /api/payments/invoice

    • Verify Payment

    Verifies whether a payment has been received at the address or LN invoice. Returns confirmations, block hash, txid, amount received, and overpayment/underpayment analysis.

  • /api/payments/verify

    • Payment Status API

    Returns live status of any given payment (pending, confirmed, settled, expired, failed). Provides refund or dispute metadata where applicable.

  • /api/payments/status

    • Lightning Payment (LN)

    Supports real-time Lightning Network payments via LNURL, BOLT11, or keysend. Endpoints auto-detect optimal route using ZZX-Labs’ active node channels.

  • /api/payments/lightning

    • Admin-only Payment Logs

    Secure access to full transaction logs, including origin IP, timestamps, asset/product tied to payment, and confirmation depths. Only accessible by internal tokens.

  • /api/payments/logs

    ZZX-Labs offers white-label deployment of our Bitcoin-only payment processor. Use our node infrastructure to power your Bitcoin-native commerce or integrate Lightning into your software stack.
