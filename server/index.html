Host Operations Policy

Rules for safe deployment documentation and clean production practice.

Public-Safe Units

Commit reusable service templates, but keep private hostnames, secrets, and environment values out of Git.

Use Environment Files

Place host-specific runtime values in `/etc/zzx-labs/*.env` and never commit those files.

Separate Logs

Write logs to controlled runtime paths, sanitize any public summaries, and avoid publishing raw logs.

Restart Cleanly

Use systemd restart policies for crawlers, APIs, timers, mirrors, and long-running host services.
